# Search strategy

Use these principles to choose experiments. `INSIGHT.md` remains the raw source transcript; this is the operational version.

## 1. Trade accuracy for speed deliberately

Start wide and relatively accurate: make several materially different measurements. Narrow only after evidence identifies promising areas. Production optimizes for completion speed, but occasional measurements should continue.

## 2. Escape local minima

Small refinements cannot reveal a better direction across a shallow region. Label cheap, high-distance experiments as leaps. Offer one after two refinement-only rounds even when the current branch appears healthy.

## 3. Search around evidence, not blindly

Use the user's prior games, reference games, reviews, and known patterns as navigation data. Avoid both direct imitation and novelty that destroys readability. Ask for reference and avoidance examples when they would change the first round.

## 4. Optimize the correct objective

Joy, portfolio value, learning, and commercial success still require different search objectives, but the studio keeps the numeric signal intentionally simple: one one-to-five `fun` rating. Use written feedback, telemetry, and agent critique to diagnose readability, appeal, fantasy fit, replayability, and scope without pretending those proxies combine into a more precise score.

Never let the fun rating silently replace the full stated objective.

## 5. Expect noisy measurements

One short session, the creator's attachment, novelty, fatigue, or a runtime defect can distort a result. Re-measure close calls, contradictions, extreme ratings, and sessions affected by errors.

## 6. Minimize exploration cost to explore more

Prototypes need speed and a useful measurement, not production architecture. Prefer primitive visuals, one hypothesis, fixed dependencies, and short sessions. Separate gameplay and presentation studies when combining them would add confounding variables.

## 7. Reduce ownership bias

When multiple people or scouts originate branches, let someone else critique or extend them. Do not compromise by combining every idea. Cross branches only around an explicit synthesis hypothesis.

## Search-health warnings

Call out these patterns:

- no completed work is ever rejected;
- work is constantly or very late rejected;
- repeated refinement occurs without a leap;
- branches change too many variables to compare;
- multiple decision-makers optimize different objectives;
- selection rests on too little or contradictory evidence.
